## 결론 및 인사이트

### 핵심 발견사항

1. **인덱스의 효과**

   - DB 쿼리 실행 시간을 크게 단축
   - 하지만 트래픽이 많으면 DB 부하가 여전히 큼
   - 캐시와 함께 사용해야 최적의 성능 달성

2. **캐시의 효과**

   - Hot/Warm 데이터 처리로 DB 부하를 크게 감소
   - 응답 시간을 극적으로 단축 (특히 p50)
   - 캐시 히트율이 높을수록 효과가 큼

3. **전략 조합의 중요성**
   - 인덱스와 캐시를 함께 사용하면 각각의 장점을 극대화
   - 인덱스: 캐시 미스 시 DB 조회 성능 최적화
   - 캐시: Hot/Warm 데이터 처리로 DB 부하 최소화
